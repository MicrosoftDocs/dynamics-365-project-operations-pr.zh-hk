# <a name="work-breakdown-structures-overview"></a>分工結構圖階層概觀

[!include [banner](../includes/banner.md)]

分工結構圖 (WBS) 說明專案將會完成的工作。 這是工作的階層，表示專案團隊對工作構成部分以及每個元件或工作大小、成本和期間的認識。 WBS 有三個主要用途：

-   以任務為單位描述工作的明細或構成部分。
-   排定專案工作。
-   估計每項任務的成本。

WBS 的詳細程度取決於估計值所需的準確度以及針對這些估計值進行追蹤所需的層級。 對排程進度落後或成本效益不彰的容錯度極低的專案，通常需要更詳細的 WBS，並努力依據 WBS 追蹤工作進度與成本。 這種專案在建築及工程產業中很常見。 

相比之下，媒體與廣告、軟體以及 IT 基礎結構等產業中的專案多半很獨特，而且生產力是看執行任務者個人的經驗和能力而定。 因此，這些產業會使用 WBS 取得專案的近似大小，而不是詳細追蹤該專案的進度。 

建立 WBS 是一項繁複程序，通常歷時甚久才能完成，而且需要各種各樣的人進行共同作業與資訊交流。 本主題說明如何使用 WBS 增強功能來您的評估與追蹤需求。

## <a name="prerequisites-for-creating-a-wbs"></a>建立 WBS 的先決條件
若要建立 WBS，您必須可以建立工作排程並估計工作成本。

### <a name="prerequisites-for-creating-a-work-schedule"></a>建立工作排程的先決條件

若要使用 WBS 功能的完整排程功能，請完成下列設定：

1.  設定預設行事曆和專案行事曆：
    1.  按一下 **專案管理與會計** &gt; **設定** &gt; **專案管理與會計參數** &gt; **排程** 。 在 **預設工作行事曆** 欄位中，指定預設行事曆。 這會是所建立的任何新專案的預設工作行事曆。
    2.  您可以變更特定專案的預設行事曆。 按一下專案的詳細資料頁面，然後在 **專案團隊與排程** 快速分頁上，選取另一個行事曆以更新 **排程行事曆** 欄位。

2.  設定標準工作天數和工作時數。 您設為專案工作行事曆的行事曆會在 WBS 中用來決定下列資訊：

-   工作天數和假日
-   一天中的工作時數

若要設定行事曆的工作天數和工作時數，或是要建立新的行事曆，請按一下 **組織管理** &gt; **一般** &gt; **行事曆** 。

### <a name="prerequisites-for-estimating-the-cost-of-work"></a>估計工作成本的先決條件

若要使用 WBS 的完整成本估計功能，您應該針對工作者、人力類別、費用和服務費以及項目來設定成本與售價。

-   若要針對人力、費用和服務費類別設定成本與售價，請按一下 **專案管理與會計** &gt; **設定** &gt; **價格** 。
-   若要設定項目的成本與售價，請針對 [產品資訊管理] 中 **已發行的產品** 清單頁面上的每個項目使用 **交易合約** 頁面。

## <a name="creating-a-wbs"></a>建立 WBS
建立 WBS 涉及三個活動：

1.  **工作分解** – 建立將工作細分成易於管理的區塊或任務的明細。
2.  **工作排程** – 估計完成任務、設定任務相互依存性所需的時間，以及選取任務的開始和結束日期。
3.  **成本估計** – 估計每項任務的成本。

下列各節討論 WBS 功能如何協助進行這些活動。

### <a name="work-decomposition"></a>工作分解

建立工作的明細或分解通常是 WBS 建立程序的第一個步驟。 WBS 功能支援工作明細或分解的下列基本建構。 

**專案根任務** ：專案根任務是專案的最上層摘要任務。 所有其他專案任務都是在其底下建立。 根任務的名稱一律設定為專案名稱。 根節點的投入量、日期和期間會歸併總結根任務底下任務的值。 您無法修改根節點的屬性，或刪除根節點。

**摘要或容器任務** ：摘要任務是其下層有子任務或組成任務的任務。 摘要任務沒有自己的工作投入量或成本。 反而是，摘要任務的工作投入量與成本是其組成任務工作投入量與成本的加總。 最早的組成任務開始日期會當做摘要任務的開始日期，而最晚的組成任務結束日期則當做結束日期。 您可以修改摘要任務的名稱，但是無法修改投入量、日期和期間的排程屬性。 如果刪除摘要任務，也會刪除其所有的組成任務。 

**分葉節點任務** ：分葉節點任務表示專案中最細微的工作封裝。 分葉節點有預估的投入量、計劃資源數目、計劃開始和結束日期以及期間。 

您可以完成下列階層作業，以啟用建立專案工作階層或分解的功能。 

**新增任務** ：您所建立的任何新任務都會自動新增至根節點之下，並將自動 WBS 編號指派給任務。 WBS 編號代表任務在階層中的層級。 專案根任務底下第一層任務會使用 1、2、3 (依此類推) 的編號配置。 第一層底下的任務使用 1.1、1.2、1.3 (依此類推) 的編號配置。 對於每個在上一層底下新增的層級，會加入新的點標記序號。 

您目前無法自訂 WBS 編號方式。 

**縮排任務** ：將任務縮排時，此任務會變成其前面任務的下層。 系統自動根據其新上層的 WBS 編號重新計算新下層任務的 WBS 編號。 上層任務現在是摘要或容器任務，因此變成其組成任務的彙總。 

> [!NOTE] 
> 將任務縮排於在此縮排作業前為分葉節點的任務底下時，新建立的摘要任務會遺失其本身的日期、投入量和資源數。 該任務現在會使用新的組成任務的值摘要。 

**凸排任務** ：將任務凸排時，此任務就不再是其上層的組成任務。 此任務的 WBS 編號會自動重新計算，以反映任務在階層中的新層級。 也會重新計算任務原先上層任務的投入量、成本和日期，以排除該任務。 

**上移和下移** ：按一下 **上移** 和 **下移** 時，您會變更任務在其上層階層中的位置。 任務的位置不會影響任務的投入量、成本、日期或期間。 不過，其任務的 WBS 編號會自動重新計算，以反映任務的新位置。

### <a name="schedule-estimation"></a>排程估計

排程估計通常是 WBS 建立作業中的第二個步驟。 依照最佳做法，您應該在建立工作之後，完成排程估計。 Finance 中的 **分工結構圖** 頁面有兩個區段。 上半部窗格用來進行排程估計，而下半部窗格則包含您可用於成本估計的 **估計成本與營收** 索引標籤。 
**任務相依性** ：在 WBS 中，您可以建立任務之間的前置任務關聯。 將前置任務指派至任務時，該任務只有在其所有前置任務皆已完成後才能開始。 任務的計劃開始日期會自動設定為其所有前置任務的最晚日期。 

**任務排程** ：下列因素決定分葉節點任務任務的排程：

-   前置任務
-   投入
-   資源數目
-   開始和結束日期

沒有前置任務的分葉節點任務的開始日期會自動設定為專案的排程開始日期。 分葉節點任務的期間一律計算為開始和結束日期之間的工作天數。 

*<strong><em>排程規則</em></strong>* ：開啟自動排程協助時，下列規則會套用至分葉節點任務的任務排程：

-   根據專案的排程行事曆，任務的開始與結束日期必須是工作日。
-   有前置任務之任務的開始日期會自動設定為其前置任務的最晚結束日期。
-   任務的投資量會依下列方式自動計算：

人數 × 期間 × 專案行事曆的標準工作日中的時數。 

在某些情況下，您可能想要偏離這些規則。 您可以關閉自動排程來防止 Finance 自動設定或更正分葉節點任務的任何屬性。 輸入造成違反任何排程規則的任務的資訊時，該任務會顯示排程錯誤圖示。 如果不希望顯示排程錯誤，請按一下 **顯示排程錯誤** 以關閉此功能。 

> [!NOTE] 
> 不論自動排程協助是開啟還是關閉，摘要或容器任務的值都會繼續計算為組成任務值的加總。 

**修正排程錯誤** ：開啟自動排程協助時，排程錯誤不太可能發生。 不過，如果您將自動排程協助關閉，稍後再重新開啟，就可能會在 WBS 中顯示排程錯誤圖示。 

**依任務修正排程錯誤** ：按兩下特定任務的排程錯誤圖示時，對話方塊會顯示該任務所有的排程錯誤。 您可以決定要修正任務的哪些排程錯誤。 

**修正所有排程錯誤** ：如果您想要 Finance 修正 WBS 中的所有排程錯誤，請按一下動作窗格中的 **修正所有排程差異** 。 

> [!NOTE] 
> 此功能可能會導致大幅修改 WBS。 錯誤會依下列順序進行更正：

1.  修改所有任務的估計投入量，使其等於專案行事曆中定義的產能。
2.  修改每個任務的開始日期，讓任務於其所有前置任務皆完成後再開始。
3.  修改每個任務的開始日期以移除前置任務開始日期的差距。

### <a name="cost-estimation"></a>成本估計

如本文稍早所述，您可以使用 **分工結構圖** 頁面下半部窗格中的 **估計成本與營收** 索引標籤，輸入每個分葉節點任務的成本估計值。 

> [!NOTE] 
> 您無法修改摘要或容器任務的成本估計值。 摘要任務的成本估計值等於其分葉節點任務的成本估計加總。 每個任務的估計總成本會計算為下列交易類型的估計成本金額加總：

-   人力
-   項目或材料
-   費用

**服務費** 交易類型是用來估計資費型營收。 此交易類型沒有成本組成部分，因此在估計成本時不納入考慮。 

**記帳** 交易類型會用來記錄固定價值類型專案中的合約規定銷售價值。 此交易類型也不會在估計成本時納入考慮。 

估計每個任務的人力、材料及費用成本時，您必須將專案類別指派至估計成本。 

**估計人力成本** ：對於每個分葉節點任務，您可以按小時並依預設類別，指派工作投入量。 因此，當您設定任務的排程時，該任務的人力成本估計值會按照人力預設類別自動加計。 此成本估計值會顯示在該任務 **明細詳細資料** 網格中的 **估計成本與營收** 索引標籤上 。 如果您需要更多人力成本估計，您可以在此索引標籤中加入。如果您在人力成本估計上增加或減少時數，就會自動重新計算任務的排程。 

**估計費用與材料成本** ：如果您需要估計值， **估計成本與營收** 索引標籤也能讓您預估任務的費用及材料成本。 

每項人力或費用估計明細的成本與售價，都取決於位在 **專案管理與會計** &gt; **設定** &gt; **定價** 的定價表中每個類別所定義的設定。 對於項目，其成本與售價預設會從 [產品資訊管理] 中 **已發行的產品** 清單頁面上的項目或交易合約中加入。

## <a name="tracking-progress-on-the-wbs"></a>追蹤 WBS 的進度
有些產業會以劃分極細微的層級依據 WBS 來追蹤專案的進度，而其他產業則以較高的 WBS 層級追蹤進度。 本節說明如何針對您的專案需求使用 WBS 追蹤功能。 

Finance 有三個適用於專案 WBS 的檢視表：規劃檢視表、投入量追蹤檢視表和成本追蹤檢視表。

### <a name="planning-view"></a>規劃檢視表

規劃檢視表會顯示排程與成本資訊的計劃或基準估計。 雖然沒有用於追蹤專案 WBS 版本與基準的功能，但是此檢視表中的值主要用來表示基準版本。 本主題的「排程估計值」及「成本估計值」小節說明此檢視表以及如何用來建立 WBS。

### <a name="effort-tracking-view"></a>努力追蹤檢視表

投入量追蹤檢視表會在 WBS 中顯示任務進度的追蹤。 這會將任務的累積實際投入時數與計劃投入時數進行比較。 下列公式提供投入量追蹤檢視表中的值：

-   進度百分比 = 目前為止的實際投入量 ÷ 任務的計劃投入量
-   剩餘投入量 (也稱為尚未完工估計值 \[ETC\]) = 計劃投入量 – 目前為止的實際投入量
-   完工估計值 (EAC) = 剩餘未完成投入量 + 目前為止實際投入量
-   預計投入量差異 = 計劃投入量 – EAC

投入量追蹤檢視表會根據 EAC 大於還是小於計劃投入量，顯示任務投入量差異的推測：

-   如果 EAC 大於計劃投入量，則推測任務需要比原先規劃還要多的時間，並且進度落後於排程。
-   如果 EAC 小於計劃投入量，則推測任務需要比原先規劃還要少的時間，並且進度較排程提前。

**專案經理對投入量的重新推測** ：專案經理或其他追蹤專案進度的角色，有時也必須修訂任務原先的估計值。 由於各種原因，任務的進展速度可能會比原先預期的更快或更慢。 例如，範圍已縮減，或工作者的經驗比原先規劃的還要少。 推測是專案經理根據專案的目前狀態，對估計值產生的看法。 一般來說，您最好不要變更基準數字，因為專案基準代表合理滿意發佈的專案排程與成本估計值文件，這是專案所有利害關係人都同意的值。 

專案經理有兩種可用來修改任務投入量的方法：

-   修改剩餘未完成投入量，這會自動設定以更新任務的實際剩餘投入量。
-   修改進度百分比，這會自動設定以更新任務的實際進度。

這兩種方法都會導致重新計算任務的 ETC、EAC 和進度百分比，以及任務的預計投入量差異。 此外，還會重新計算摘要任務的 EAC、ETC 和進度百分比，並更新其預計投入量差異。 

**已修改的摘要任務投入量** ：您可以修改摘要或容器任務上的投入量。 不論您修改這些值所用的是摘要任務的剩餘未完成投入量還是進度百分比，都會依下列順序自動進行計算：

1.  計算工作的 EAC、ETC 和進度百分比。
2.  按照與原先 EAC 金額相同的比例，將新的 EAC 分佈至下層任務。
3.  計算每個分葉節點任務的新 EAC。
4.  根據新的 EAC 值，重新計算所有受影響下層任務的剩餘未完成投入量和進度百分比。 也會重新計算任務的投入量差異。
5.  此外，還會從分葉節點重新計算摘要任務的 EAC。

按一下投入量追蹤檢視表中的 **展開至層級** ，以設定要追蹤和維護 WBS 所到的層級。 每當您開啟 WBS 時，它都會自動展開至投入量追蹤檢視表中的那個層級。

### <a name="cost-tracking-view"></a>成本追蹤檢視表

成本追蹤檢視表會顯示任務成本耗用的追蹤。 在此檢視表中，會將目前為止所花費的實際成本與任務的計劃成本進行比較。 下列公式提供成本追蹤檢視表中的值：

-   成本耗用百分比 = 目前為止的實際成本 ÷ 任務的計劃成本
-   尚未完工成本 (CTC) = 計劃成本 – 目前為止的實際成本
-   完工估計值 (EAC) = CTC + 目前為止的實際成本
-   預計成本差異 = 計劃成本 – EAC

成本追蹤檢視表會根據 EAC 大於還是小於計劃成本，顯示任務成本差異的推測：

-   如果 EAC 大於計劃成本，則推測任務使用比原先規劃還要多的金額，並且超出預算。
-   如果 EAC 小於計劃成本，則推測任務使用比原先規劃還要少的金額，並且在預算之內。

**專案經理對成本的重新推測** ：專案經理必須使用 CTC 來修訂任務的原先的成本估計。 專案經理可以將 CTC 值修改為完成任務所需的成本。 如果修改 CTC 值，就會重新計算任務的 CTC、EAC 和成本耗用百分比，以及預計成本差異。 此外，還會重新計算摘要任務的 EAC、ETC 和成本耗用百分比，並更新其預計成本差異。 

> [!NOTE] 
> 在投入量追蹤檢視表中修訂 WBS 任務的投入量時，任務的 CTC、EAC、成本耗用百分比和預計成本差異都會在成本追蹤檢視表中進行重新計算。 不過，成本修訂不會影響投入量追蹤檢視表中的值，因為依交易類型 (人力、材料或費用) 區分的成本或是專案類別並未修訂。 

**摘要任務的成本推測修訂** ：您可以修訂摘要任務的成本，而計算會依下列順序自動進行。

1.  重新計算任務的 EAC、CTC 和成本耗用百分比。
2.  按照與任務原先 EAC 相同的比例，將新的 EAC 分佈至下層任務。
3.  計算每個任務的新 EAC。
4.  根據 EAC 值，重新計算受影響下層任務的 CTS 和成本耗用百分比。 也會重新計算任務的成本差異。
5.  根據此變更重新計算所有摘要任務的 EAC。

按一下成本追蹤檢視表中的 **展開至層級** ，以設定要追蹤和維護 WBS 所到的層級。 每當您開啟 WBS 時，它都會展開至成本追蹤檢視表中的那個層級。

### <a name="earned-value-management"></a>實獲值管理

您可以使用實獲值分析方法 (EVM) 來追蹤專案的進度。 您可以在專案經理的角色中心查看實獲值計量。 實獲值圖表元件會顯示計劃值與實際成本的分時期值。 截至目前日期的實獲值會顯示為點。 實獲值的分時期資料目前無法使用。 

實獲值圖表上的時間階段會依週或依月顯示。 本節說明 EVM 的三大支柱：計劃值、實獲值和實際成本。 

**計劃值** ：EVM 理論指出計劃值圖表呈現的是專案團隊所規劃要在專案上贏得產值的比率。 

Finance 會在繪製計劃值時使用 0:100 獲利規則。 依據此規則，任務的值會在其結束日期當時開始轉記至該任務。 完成 100% 任務之前，不會轉記任何值。 

在 [專案管理與會計] 中，您可以輸入分葉節點的結束日期及其計劃成本。 當計劃值圖表依週顯示時，此圖表會按照週歸併總結所有分葉節點任務在專案期間的計劃值。 

**實獲值** ：EVM 理論指出實獲值圖表呈現的是專案團隊在專案上實際贏得產值的比率。 

Finance 會在繪製實獲值時使用 0:100 獲利規則。 依據此規則，任務的值會在其結束日期當時開始轉記至該任務。 完成 100% 任務之前，不會轉記任何值。 

計算實獲值時，會考慮每個任務的進度百分比。 依照 0:100 獲利規則，只有在指定期間完成的工作，才要考慮計算截至該期間結束時的實獲值。 專案的實獲值是在建立圖表時針對所有已完成的任務進行計算。 

> [!NOTE] 
> WBS 追蹤系統目前沒有資料結構，無法儲存每個任務上的歷史進度百分比。 因此，只能從處理 Cube 的時候開始報告實獲值。 定期處理 Cube 以更新角色中心顯示的實獲值資料。 

**實際成本** ：EVM 理論指出實際成本圖呈現的是專案上所花費資金的比率。 

過帳至專案的交易會用來繪製實際成本明細。 成本是依照日期歸併總結。 此資料接著會在實獲值圖表上用來繪製依週或依月區分的實際成本。

### <a name="how-to-use-the-concepts-of-planned-value-earned-value-and-actual-cost"></a>如何運用計劃值、實獲值和實際成本的概念

**排程差異** ：進行規劃時，您可以在時間表上建立工作預測。 因此，計劃值是專案規劃人員認為工作會在專案中完成的比率。 專案開始進行後，當工作完成時，專案便會贏得產值。 您可以比較計劃值與實獲值，以檢視專案工作的進展情況。 這項比較的結果稱為排程差異。 

如果一段期間的計劃值大於實獲值，表示專案所完成的工作量比已規劃的還要少。 因此，專案進度落後於排程。 計劃值和實獲值表示為貨幣形式，因此專案的延遲時間也會以貨幣值來指定。 

如果一段期間的計劃值小於實獲值，表示專案所完成的工作量比已規劃的還要多。 因此，專案進度較排程提前。 此提前時間也是以貨幣值來指定。

**成本差異** ：實獲值會使用成本價做為乘數，因此實獲值表示專案所完成工作的成本。 隨著專案進展，交易記錄會提供該專案實際花費的資金記錄。 您可以將實獲值與實際成本做比較，檢視所花費資金相對於所獲得產值的金額。 這項比較的結果稱為成本差異。 

如果一段期間所花費的實際成本大於實獲值，表示所花費的資金比所獲得的還要多。 因此，專案超出預算。 

如果一段期間所花費的實際成本小於實獲值，表示所獲得的資金比所花費的還要多。 因此，專案還在預算之內。

## <a name="wbs-templates"></a>WBS 範本
您可以使用 WBS 範本功能來建立專案的標準範本。 如果您的公司提供的專案包含大量可重複的工作，您應該考慮建立 WBS 範本。 

您可以從現有專案的 WBS 建立 WBS 範本，這樣就可以在日後的類似專案上重複使用該專案規劃期間收集到的知識和最佳做法。 不過，將整個 WBS 儲存為範本有時可能沒有意義。 因此，您也可以從專案 WBS 的某些部分建立範本。

### <a name="saving-a-projects-wbs-as-a-template"></a>將專案的 WBS 儲存為範本

建立範本後，您可以在根節點底下或專案 WBS 中任何任務底下將其匯入至新專案的 WBS。

### <a name="importing-a-wbs-template-into-a-projects-wbs"></a>將 WBS 範本匯入至專案的 WBS

匯入任務時，範本中的任務是根據匯入這些任務所到任務的開始日期來組織。 匯入期間，範本工作的前置任務關聯會使用來計算所匯入任務的開始日期。 目的地專案的標準工作行事曆會套用來計算所匯入任務的結束日期，這樣就會保留在目前專案工作行事曆中定義的工作天數和標準工作時數。 

估計明細的成本金額和售價會套用來確保專案或專案合約特定的價格具有有效日期。

### <a name="differences-between-a-projects-wbs-and-a-wbs-template"></a>專案 WBS 與 WBS 範本之間的區別

-   WBS 範本中的任務沒有開始日期和結束日期。

WBS 範本沒有設定工作時數和非工作日數。

-   WBS 範本永遠使用設定為所有專案預設行事曆的排程行事曆。

預設排程行事曆僅用於了解標準工作日中的時數。

-   前置任務關聯不會複製到 WBS 範本。

由於 WBS 範本沒有日期，因此不需要以前置任務結束日期為依據的開始日期邏輯。

-   在 WBS 範本中建立任務時，系統會自動建立人力成本估計明細。 售價和成本金額是從所選工作者複製而來。

可以手動建立費用和項目成本，就像可以在專案的 WBS 中建立一樣。

-   與下列公式有偏差時，會顯示排程錯誤：

投入量 = 資源數目 × 期間 × 標準工作日中的時數 

您可以按一下 **修正所有排程錯誤** ，同時更正所有排程錯誤。 

或者，也可以按一下每個任務的警告圖示，個別更正排程錯誤。
